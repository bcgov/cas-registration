1. Create Client
	•	Endpoint: /ws/finance/client
	•	Method: POST
	•	Description: Creates a new client and one associated address. Returns the ClientObjectId created in the system.
	•	Assumption: Addresses are located in Canada/USA only.

Headers:
Authorization: Bearer

Request Body:
{
  "clientGUID": "string (40 max)",  // Placeholder for BCIERS unique client identifier
  "companyName": "string (150 max)",  // Required if LastName is blank
  "lastName": "string (60 max)",      // Required if CompanyName is blank
  "doingBusinessAs": "string (100 max)",
  "firstName": "string (60 max)",
  "middleName": "string (60 max)",
  "title": "string (100 max)",
  "businessPhone": "number (10 digits)",
  "homeNumber": "number (10 digits)",
  "cellularNumber": "number (10 digits)",
  "faxNumber": "number (10 digits)",
  "businessPhoneExt": "number (4 digits)", // Only if BusinessPhone is provided
  // One of the following two parameters are required if CompanyName is provided
  "bcCompanyRegistrationNumber": "string (25 max)",
  "bcCompanySocietyNumber": "string (25 max)",
  "email": "string (60 max)",
  "dateOfBirth": "YYYY-MM-DD",
  "addressLine1": "string (150 max)", // Mandatory
  "addressLine2": "string (100 max)",
  "city": "string (50 max)", // Mandatory
  "stateProvince": "string (2 max)", // Mandatory
  "postalCode": "string (10 max)", // Mandatory
  "country": "string (30 max)" // If not specified, assumed as Canada
}

Success Response (200):
{
  "clientObjectId": "number",  // newly created ClientObjectId
  "clientGUID": "string"
}

Business Rules:
	•	Pass either companyName or lastName, but not both.
	•	Provide at least one of the following: businessPhone, homeNumber, cellularNumber, or faxNumber.
	•	businessPhoneExt is allowed only if businessPhone is provided.
	•	Provide at least one of bcCompanyRegistrationNumber or bcCompanySocietyNumber if companyName is provided.
	•	addressLine1, city, stateProvince, and postalCode are mandatory.

4. Query Balance
	•	Endpoint: /ws/finance/client/<clientId>/balance
	•	Method: GET
	•	Description: Returns the outstanding balance for an existing client. clientId can be either the client object id or client guid.
Headers:
Authorization: Bearer

Query Parameters:
	•	BalanceType
	•	“regular” will return outstanding regular balance
	•	“interest” will return outstanding interest balance
	•	“nsf” will return outstanding NSF balance
	•	“outstanding” will return Regular+Interest+NSF outstanding balance
	•	“prepayment” will return the balance in the client’s pre-payment account

Example:
GET /ws/finance/client/12345/balance?BalanceType=nsf
Authorization: Bearer

Success Response:
{
  "clientObjectId": "number",
  "clientGUID": "string",
  "balanceType": "string",
  "outstandingBalance": "number"
}

6. Adjust Fees
	•	Endpoint: /ws/finance/client/<clientId>/adjust
	•	Method: POST
	•	Description: Adjusts fee records for a specified client. clientId can be either the client object id or client guid.
Headers:
Authorization: Bearer
Request Body:
{
  "adjustments": [
    {
      "feeGUID": "string",  // Either FeeGUID OR FeeObjectId must be specified
      "feeObjectId": "number"
      "adjustmentGUID": "string (40 max)"  // BCIERS unique adjustment identifier
      "adjustmentTotal": "number",  // Mandatory (ex. -1140.00)
      "date": "YYYY-MM-DD",  // If not passed, will default to Fee Date
      "reason": "string (4000 max)",  // Mandatory
      "type": "string (17 max)",  // must be "Fee Recalculation" OR "Adjustment"
      "comment": "string (4000 max)"
    }
  ]
}
Success Response:
{
  "clientObjectId": "string",
  "clientGUID": "string",
  "adjustments": [
    {
      "feeGUID": "string",
      "feeObjectId": "number",
      "adjustmentGUID": "string",
      "adjustmentObjectId": "number"  // newly created AdjustmentObjectId
    }
  ]
}

9. Create Invoice
	•	Endpoint: /ws/finance/client/<clientId>/invoice
	•	Method: POST
	•	Description: Create an Invoice for a set of fees related to a specific client.
Headers:
Authorization: Bearer

Example:
POST /api/client/12345/invoice
Request Body:
{
  "paymentDueDate": "YYYY-MM-DD", 
  "businessAreaCode": "string",
  "fees": ["feeId1", "feeId2", "feeId3", …]
}
	•	A feeId can be the feeObjectId or feeGUID for a specific fee.
Example Success Response:
{
  "clientObjectId": "number",
  "businessAreaCode": "string",
  "clientGUID": "string",
  "invoiceNumber": "string"
}
